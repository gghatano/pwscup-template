# Task-029: チュートリアル・デモコンテンツ作成

## 概要
初学者向けのチュートリアルドキュメントとデモスクリプトを作成する。
コンテストの概念理解から、実際のデータ操作・評価・スコア確認までを一通り体験できる内容。

## 依存タスク
- Phase 01-06 すべて完了済み

## 成果物

### 1. docs/tutorial.md
初学者向けステップバイステップガイド。以下の構成：

#### Step 0: 環境構築
- uv のインストール
- プロジェクトのセットアップ（uv sync）

#### Step 1: コンテストの仕組みを理解する
- 匿名化部門と再識別部門の説明
- スコアリングの考え方（有用性×安全性）
- データの構造（QI, SA, 非機微属性）

#### Step 2: サンプルデータを見てみる
- schema.json の読み方
- sample_original.csv の中身確認
- 各カラムの役割を理解

#### Step 3: ベースライン匿名化を試す
- examples/anonymize_example の説明
- ローカル評価コマンドで実行
- 有用性・安全性スコアの読み方

#### Step 4: ベースライン再識別を試す
- examples/reidentify_example の説明
- 再識別評価の実行
- precision/recall の意味

#### Step 5: スコアを改善してみよう
- 匿名化の改善ヒント（k値を上げる、汎化戦略）
- 再識別の改善ヒント（特徴量の活用）
- トレードオフの理解

#### Step 6: 提出の流れ
- submit コマンドの使い方
- leaderboard の見方
- status で履歴確認

### 2. scripts/run_demo.py
パイプライン全体を1コマンドで体験できるデモスクリプト：
- サンプルデータの確認
- ベースライン匿名化の実行と評価
- ベースライン再識別の実行と評価
- スコア表示
- 各ステップで分かりやすい日本語解説を出力

## 完了条件
- docs/tutorial.md が作成され、手順通りに実行可能
- scripts/run_demo.py が `uv run python scripts/run_demo.py` で動作
- 既存テストが壊れないこと
