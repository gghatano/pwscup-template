# Task-024: E2Eテスト

## 概要
提出→評価→スコア算出→リーダーボード表示の一連フローのE2Eテスト

## 依存タスク
- task-018, task-022, task-023

## 成果物
- `tests/test_e2e/test_full_flow.py`

## 詳細

### テストシナリオ

#### シナリオ1: 匿名化提出→ローカル評価
1. サンプルデータを用意
2. ベースライン匿名化を `pwscup evaluate anonymize` で評価
3. 有用性・安全性スコアが0〜1の範囲で返る
4. 最低基準(k≧2)を満たす

#### シナリオ2: 再識別提出→ローカル評価
1. シナリオ1で生成された匿名化データを使用
2. ベースライン再識別を `pwscup evaluate reidentify` で評価
3. precision, recall が0〜1の範囲で返る

#### シナリオ3: submit→DB保存→leaderboard
1. 2チーム分の匿名化を submit
2. 2チーム分の再識別を submit
3. leaderboard で3つの部門の順位表示を確認
4. status で提出履歴を確認

#### シナリオ4: エラーケース
- k < 2 の匿名化提出が拒否される
- タイムアウトするアルゴリズムが適切に処理される
- 不正なフォーマットの提出が拒否される

## 完了条件
- 全シナリオが自動テストとしてパスする
- CIで実行可能
