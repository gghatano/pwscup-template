# Task-006: 合成データ生成スクリプト

## 概要
コンテスト用の合成データ（生データ）を生成するスクリプトの作成

## 依存タスク
- task-005

## 成果物
- `scripts/generate_data.py` — 合成データ生成スクリプト
- `data/original/` — 生成された生データ（Git管理外）

## 詳細

### データ設計
仕様書 Section 2.2〜2.3 に基づく。以下の属性を含む個人レコードテーブル：

| カラム | 型 | 値域 | 属性分類 |
|--------|------|------|----------|
| id | int | 1〜N | 識別子 |
| age | int | 18〜90 | 準識別子 |
| gender | str | M/F/Other | 準識別子 |
| zipcode | str | 100-0001〜999-9999 | 準識別子 |
| occupation | str | 20種程度 | 準識別子 |
| education | str | 高卒/専門/学士/修士/博士 | 準識別子 |
| disease | str | 15種程度 | 機微属性 |
| salary | int | 200万〜2000万 | 機微属性 |
| hobby | str | 20種程度 | 非機微属性 |

### 生成要件
- レコード数: 10,000件（サンプル）/ 30,000件（予選）/ 50,000件（本選）
- 現実的な相関構造を持つこと（例: 年齢と年収の正の相関、学歴と職業の関連）
- 属性間の条件付き分布を定義して生成する
- 乱数シードを固定して再現可能にする

### 出力
- CSV形式（UTF-8）
- `data/original/sample.csv`, `data/original/qualifying.csv`, `data/original/final.csv`

## 完了条件
- スクリプトを実行して3種類のデータが生成できる
- 生成データがschema.jsonに適合する
- 基本統計量（分布、相関）が妥当である
