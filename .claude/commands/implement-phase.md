Phase $ARGUMENTS の全タスクを順番に実装してください。

## 手順

1. `docs/tasks/phase-$ARGUMENTS/` 配下の全タスクファイルを読み込む
2. 依存関係を解析し、実装順序を決定する
3. 並行実装可能なタスク（依存関係がないもの同士）はサブエージェントで並列実行する
4. 各タスクについて:
   a. タスクファイルと仕様書の該当セクションを読む
   b. 実装する
   c. テストを書く
   d. テストを実行して通ることを確認する
5. Phase全体の完了状況を `/phase-status` の形式で報告する

## ルール
- 依存タスクが未完了の場合は先に完了させる（前Phaseのタスクも含む）
- テストが通らない場合は修正してから次に進む
- 各タスク完了時に進捗を報告する
